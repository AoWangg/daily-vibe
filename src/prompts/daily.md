# 开发日报生成

你是资深研发 TL。根据以下按时间排序的工作片段，生成当日【开发日报】：

## 工作片段
{sessions}

## 生成要求
- **概览**：做了哪些任务/模块，涉及哪些文件/提交
- **关键产出**：功能/脚手架/脚本/PR（如有链接则列出）
- **运行/测试**：跑过哪些命令，成功与失败
- **待办与阻塞**：下一步计划与风险

## 格式要求
- 输出 Markdown 格式
- 字数控制在 ≤ 300 行
- 使用二级/三级标题与清单
- 代码和命令使用代码块格式

## 示例结构
```markdown
# 开发日报 - {date}

## 📊 概览
- 主要工作：...
- 涉及模块：...
- 文件变更：...

## 🚀 关键产出
- [ ] 完成功能：...
- [ ] 修复问题：...
- [ ] 脚本工具：...

## 🔧 运行测试
### 成功执行
- `command1`：结果说明
- `command2`：结果说明

### 失败排查
- `error_command`：错误原因和解决方案

## 📋 待办事项
- [ ] 下一步计划
- [ ] 已知问题
- [ ] 风险点
```
